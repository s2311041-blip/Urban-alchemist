## サイズ調整仕様

### スケール範囲
- X / Y / Z 各軸独立: 最小 `0.1` 〜 最大 `3.0`（ステップ `0.05`）
- スライダーまたは3Dギズモドラッグで調整

### 3Dギズモ（Gizmo3D）
- 6方向の矢印ハンドル（+X, -X, +Y, -Y, +Z, -Z）
- 各矢印をポインターダウンしてドラッグすると、ドラッグ方向に片側伸縮する
- ブロックの回転角を考慮してローカル座標系でドラッグ計算する
- 色コード: 赤（X軸）、緑（Y軸）、青（Z軸）

### サイズ調整履歴
- Undo（戻る）/ Redo（進む）が最大30件保持される
- スタジオモード・サイズ調整モードで独立した履歴スタックを管理

## カメラ仕様

### 視点モード

| モード | 説明 | minDistance | maxDistance |
|--------|------|------------|------------|
| TPS (三人称) | アバター背後追従 | 3.5 | 20 |
| 建築モード | 俯瞰カメラ、W/A/S/D移動 | 2 | 60 |
| 神（God）視点 | 島拡張演出時の固定俯瞰 | — | — |
| スタジオ | 対象ブロックにフォーカス | — | — |

### ズーム速度
- 通常プレイ中（TPS）: `zoomSpeed = 0.22`（スクロールは細かく）、距離 1.75〜48
- 建築モード中: `zoomSpeed = 0.35`（微調整しやすいマイルドな感度）

### CameraRig（カメラ制御コンポーネント）
- `App.jsx` の `<Canvas>` 直下に常時マウントされる（シーン切り替えで消えない）
- スタジオ（`isDesigningInStudio` / `isEditingInStudio`）終了を検知し、自動でアバター背後にカメラをリセットする
- ズーム・回転: OrbitControls が担当
- 建築モードのキー移動: `GameCameraPanController` が担当

## 範囲選択・操作仕様

- `Shift + S` または下部パネルから「範囲選択」を起動
- 1点目クリック → ↑↓キーで高さ範囲調整 → 2点目クリックで矩形選択確定
- 選択後: **コピー / 移動 / 削除** の3操作が可能
- コピー・移動後の配置先は、クリックした場所で決定する

---

## 斜めブロック設計仕様

- 形状: 立方体の20箇所のアンカーポイントから2点を選んで梁を定義する
- アンカーポイントの内訳: 頂点8点 ＋ X辺中点4点 ＋ Y辺中点4点 ＋ Z辺中点4点
- 1点目クリック → 2点目クリックで梁の向き・長さが確定する
- 過去に設計した斜め形状は最大5件まで履歴として保存され、クリックで再利用できる
- 太さはXスケールおよびYスケールに比例する（デフォルト thickness = scale * 0.05）

## 状態管理（Zustand Store）仕様

### 主要な状態変数

| 変数名 | 型 | 説明 |
|--------|-----|------|
| `placedBlocks` | Block[] | 島に配置された全ブロック一覧 |
| `selectedShape` | string | 現在選択中の形状 |
| `selectedMaterial` | string | 現在選択中の素材 |
| `selectedScale` | [x,y,z] | 現在選択中のサイズ |
| `blockRotation` | number | ブロックの回転角（度） |
| `buildMode` | string\|null | 建築モードID（フリーまたはバグID） |
| `viewMode` | `'tps'\|'god'` | カメラ視点モード |
| `isDesigningInStudio` | boolean | 斜め設計スタジオ表示中 |
| `isEditingInStudio` | boolean | 3D形状スタジオ表示中 |
| `isAdjustingSize` | boolean | サイズ調整プレビュー中 |
| `bugs` | Bug[] | バグ（住民の不満）一覧 |
| `quests` | Quest[] | クエスト（解決済みの声）一覧 |
| `islandChunks` | Chunk[] | 島のチャンク一覧 |
| `recentDiagonals` | Points[][] | 斜め形状の履歴（最大5件） |
| `favorites` | Favorite[] | お気に入りブロック（最大3件） |

### 自動永続化

```javascript
// 状態変化のたびに localStorage に自動保存
useGameStore.subscribe((state) => {
  localStorage.setItem('urban_alchemist_save_v1', JSON.stringify({
    bugs, quests, islandChunks, placedBlocks, recentDiagonals
  }));
});
```

### ロード時バリデーション
- ロード時に `pos`, `scale`, `size`, `diagonalPoints` のすべての数値が `Number.isFinite` であるか厳密に検査する
- 破損・不正値が検出された場合は自動排除または安全なデフォルト値に補正する
- これにより、不正データによるリロード時のクラッシュを完全に防止している

## Undo / Redo 仕様

### 全体建築 Undo/Redo
- `placedBlocks` の全スナップショットを最大30件保持
- Ctrl+Z / Cmd+Z でUndoが可能

### スタジオ内編集 Undo/Redo
- スタジオの `scale`, `offset`, `material`, `shape` を履歴管理
- ギズモドラッグ・スライダー操作後に自動でプッシュ

### サイズ調整 Undo/Redo
- サイズ調整モードの `scale`, `offset` を独立した履歴スタックで管理

## バグ（住民の不満）データ構造

```json
{
  "id": "string",
  "pos": [x, y, z],
  "type": "dark | danger | dirty | lonely | line_sign_confusion | area_maintenance_gap | line_step_gap | area_isolation",
  "solved": false,
  "demographic": "20代 など",
  "comment": "住民のコメント文",
  "photo": "画像URL（オプション）",
  "scale": "point | line | area",
  "factor": "hard | soft | human",
  "needType": "P | V | I | M | R | L | S | C",
  "tagLabel": "#局所的欠損_障壁 など",
  "allowedPlans": ["hard_fix", "detour_path", "sign_info"],
  "chosenPlan": "hard_fix（未選択時は null）",
  "affectedGroups": ["高齢者", "車いす" など],
  "timeTag": "朝 | 昼 | 夕方 | 夜 | 常時",
  "severity": "low | mid | high"
}
```

### 代表的な解決プラン（plan）
- `lighting`（街灯）
- `hard_fix`（階段/ハーフで段差改善）
- `detour_path`（歩道の連続）
- `maintenance`（ベンチ + 歩道）
- `sign_info`（案内看板）
- `care_point`（ベンチ + 街灯）
