## 1. 視覚原則

1. **ジオラマ** — 浮遊島が模型のように一望できる
2. **光る物語** — オーブ・暗路地・街灯がクエストマーカーとして語る
3. **ブロックの喜び** — 鮮やか・読みやすい色
4. **やさしいファンタジー** — リアルすぎず子どもっぽすぎない

### 概念図とゲームが「違って見える」主因

| 概念図 | ゲーム（現状） | 近づけるには |
|--------|----------------|--------------|
| 一枚絵のイラスト（レンダー済み） | リアルタイム3D | ポストプロセス・固定カメラ |
| 固定アイソメ（平行投影） | パースカメラ | 神ビューで Orthographic 化（Phase 6） |
| 全要素が同じボクセル密度 | プロシージャル複合メッシュ | placeShapes 全面ボクセル化 |
| 暖冷のゾーン照明が極端 | 均一な太陽光 | 路地=寒色 / 広場=暖色ライト（一部実装済） |
| UIが画像に焼き込み | HTMLオーバーレイ | UIトークン統一（Phase 5） |

**正直な線引き:** 概念図100%一致は「ゲームエンジン」ではなく「宣伝用レンダラー」に近い。デモ用スクショ品質なら **固定カメラ + 強いBloom + 暖色ゾーン** で 70–80% は狙える。
