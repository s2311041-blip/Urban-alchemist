## 1. 全体構成

| 装置 | URL | エントリ | 役割 |
|------|-----|----------|------|
| **AR投稿アプリ** | `/ar.html`（`/ar` リライト） | `src/ar/main.jsx` → `ArPostingApp.jsx` | 現地で困りごと・良い場所を記録し、Supabase 等に保存 |
| **シリアスゲーム** | `/`（`index.html`） | `src/App.jsx` | 3D 島上で不満を追体験 → プラン選択 → DIY 改善 → 島拡張 |

**接続の流れ:**

```
/ar.html で Bad 投稿
  → Supabase（または JSON エクスポート）
  → ゲーム「クエストボード」で取り込み
  → 島上に場所セット（L1）+ 不満 accent（L2）+ 黒オーブ
  → タップ → プラン選択 → 改善予算内 DIY → 完成 → 島拡張
```

### 2.2 新規投稿フロー（全体）

```
① 場所を決める (place)
    ↓
② 撮影の準備 (captureIntro)
    ↓
③ 写真撮影 (capture)
    ↓
④ 設問フォーム (form) — 3〜4 ステップ
    ↓
⑤ 保存アニメ (stick) → ホーム
```

**編集時**は **④ フォームから開始**（場所・撮影・stick をスキップ）。

**実装:** `src/ar/components/ArPostFlow.jsx`

### 2.3 ① 場所を決める

#### 3 つの配置モード

| モード | UI ラベル | 仕組み | `placementMode` |
|--------|-----------|--------|-------------------|
| **feet** | 現在地 | 立っている GPS 位置にピン | `feet` |
| **map** | 地図 | Leaflet 地図でタップ／ドラッグ | `map` |
| **camera** | カメラ | 画面タップ＋方角・ピッチから約 8m 先を推定 | `camera` |

デフォルトモード: `feet`

#### GPS 精度ゲート（feet モード）

| 精度 (m) | 判定 | 配置可 |
|----------|------|--------|
| ≤12 | 良好 | ○ |
| ≤20 | 使える | ○ |
| ≤35 | やや低い | △（上書き可） |
| >35 | 低い | ×（上書き可） |

「低精度のまま続行」で fair/poor も許可。

**実装:** `src/ar/utils/gpsAccuracy.js`, `ArGpsAccuracyPanel.jsx`

#### 地理制限

- **江東区ロックは廃止**（`003_remove_koto_bounds.sql`）。有効な緯度経度なら全世界 OK
- 地図の初期中心: 江東区 `35.653, 139.826`（`KOTO_CENTER`、フォールバック用）

#### 自動取得メタデータ

| フィールド | 内容 |
|-----------|------|
| `worldPin` | 投稿位置（lat/lng） |
| `authorGeo` | 撮影者 GPS |
| `capturePose` | 方角・ピッチ等 |
| `screenTap` | 画面タップ座標 |
| `distanceM` | 推定距離 |

#### Step 2（bad）：困りごとと場所

##### 困りごとの型（needType）— 8 種

PPS 4 グループ UI（`src/ar/constants/needTypeGroups.js` + `ArNeedTypePicker.jsx`）:

| グループ | needType | ラベル | hint |
|----------|----------|--------|------|
| 行き来・移動 🚶 | P | 歩きにくい | 段差・狭さ・通りづらさ |
| | L | 行き来しにくい | 移動手段・アクセスの不足 |
| | I | 迷いやすい | 案内不足・分岐が分かりにくい |
| 滞在・居場所 ☕ | R | 休みにくい | 座る・待つ場所が足りない |
| 環境・維持 🧹 | M | 汚れ・荒れ | 清掃・維持が追いつかない |
| | V | 見えにくい | 暗い・死角で見えない ※ |
| 安心・社会 🤝 | S | こわい・不安 | 人通り・雰囲気で安心できない ※ |
| | C | 頼れる場がない | 支援・見守りに繋がりにくい |

※ V/S 境界はピッカー用 hint で明確化

| id | ラベル | ゲーム側プリセット |
|----|--------|-------------------|
| `station` | 駅（改札・ホーム） | `station_layout` 等 |
| `plaza` | 駅前・広場 | `plaza_layout` 等 |
| `bus_stop` | バス停 | `bus_stop_layout` |
| `waterfront` | 水辺・運河沿い | waterfront |
| `road` | 歩道・道路 | road |
| `commerce` | 商業施設・店舗街 | commerce |
| `park` | 公園・広場 | park |
| `lane` | 路地・住宅街 | lane |
| `none` | どれにも当てはまらない | オーブのみ |

#### Step 3（bad）：誰にとって？

##### 影響を受ける人（affectedGroups）— 複数選択可

**実装:** `src/ar/constants/arTargetGroups.js`

| id | ラベル | 備考 |
|----|--------|------|
| all | みんな | 単独選択（他と排他） |
| senior | 高齢者 | |
| wheelchair | 車いす | |
| stroller | ベビーカー | |
| child | 子ども | |
| other | その他 | 入力欄が出る |

**トグルルール:**

- 「みんな」選択 → 他すべて解除 + `affectedOther` クリア
- 他を選択 → 「みんな」解除
- 「その他」解除 → `affectedOther` クリア

**「その他」選択時:** `affectedOther`（30 字以内、`AFFECTED_OTHER_MAX_LEN`）必須

**UI 上は任意** — `canProceedStep` 上、1 人も選ばず次へ進める

##### 折りたたみ「詳しく（時間帯・深刻度）」— 任意

**時間帯 (timeTag)** — デフォルト `always`（常時）

| id | ラベル |
|----|--------|
| morning | 朝 |
| day | 昼 |
| evening | 夕方 |
| night | 夜 |
| always | 常時 |

**深刻度 (severity)** — デフォルト `mid`

| id | ラベル |
|----|--------|
| low | 軽い |
| mid | 中くらい |
| high | 深刻 |

### 2.6 保存・バリデーション

#### 送信時バリデーション（`postValidation.js`）

| 項目 | ルール |
|------|--------|
| `worldPin` | 必須（新規投稿 UI） |
| 座標 | lat −90..90, lng −180..180 |
| `comment` | 1〜500 字 |
| `photo` | あれば data URL ≤ 約 2.5MB |

#### 保存先

| 優先 | 方式 | ストレージ |
|------|------|-----------|
| 1 | Supabase | `ar_annotations` テーブル + Storage |
| 2 | LAN REST | `VITE_AR_API_URL/annotations` |
| 3 | ローカル | `localStorage` `urban_alchemist_ar_local_pool` |

#### Supabase スキーマ（`001_ar_annotations.sql`）

- テーブル `ar_annotations`（id, author_id, world_pin_lat/lng, photo_path, payload JSONB）
- 匿名 Auth（`signInAnonymously`）
- レート制限: **1 時間 10 件** / ユーザー
- RLS: 全員 read、自分のみ insert/update/delete

#### いいね（`004_ar_likes.sql` + `likesClient.js`）

| ルール | 内容 |
|--------|------|
| コスト | 1 pt / 1 回 |
| 対象 | 他人の投稿のみ |
| 重複 | 不可 |
| ローカル | `urban_alchemist_ar_like_records` |
| クラウド | `ar_likes` テーブル |

### 2.7 ポイント（pt）

**実装:** `src/ar/utils/normalizeAnnotation.js` → `computePoints()`

| 条件 | pt |
|------|-----|
| 基本 | +10 |
| コメント 20 字以上 | +5 |
| `worldPin` あり | +5 |
| 困りごと (bad) | +3 |

**最大 23 pt**（bad・長文・位置あり）。**編集時は pt 再計算なし**（`pointsAwarded` 保持）。

### 3.3 不満の分類体系

**実装:** `src/constants/barrierData.js`

#### 3×3 土台（scale × factor）

| scale | 意味 | UI ラベル | 半径倍率 |
|-------|------|-----------|----------|
| point | 一点 | 一点の問題 | ×1（基準 6m） |
| line | 動線 | 動線の問題 | ×1.35（≈8m） |
| area | 面 | 周辺エリアの問題 | ×1.8（≈11m） |

| factor | 意味 |
|--------|------|
| hard | 物理的 |
| soft | 環境・情報 |
| human | 心理的・社会的 |

#### 現行アクティブ type（8 種）

`danger`, `dirty`, `dark`, `lonely`, `line_sign_confusion`, `area_maintenance_gap`, `line_step_gap`, `area_isolation`

#### 解決プラン（7 種）

| planId | ラベル |
|--------|--------|
| `lighting` | 照明を増やす |
| `hard_fix` | 段差を解消する |
| `detour_path` | 迂回ルートを作る |
| `transit_link` | 海列車をつなぐ |
| `maintenance` | 維持しやすい空間にする |
| `sign_info` | 案内で誘導する |
| `care_point` | 見守り拠点をつくる |

#### type ごとの allowedPlans（例）

| type | allowedPlans | defaultPlan |
|------|--------------|-------------|
| danger (P) | hard_fix, detour_path, sign_info | hard_fix |
| dark (V) | lighting, detour_path, care_point | lighting |
| dirty (M) | maintenance, hard_fix, care_point | maintenance |
| lonely (S) | care_point, sign_info | care_point |
| line_sign_confusion (I) | sign_info, maintenance | sign_info |
| line_step_gap (L) | hard_fix, detour_path, transit_link | detour_path |
| area_maintenance_gap (M) | maintenance, sign_info | maintenance |
| area_isolation (C) | care_point, sign_info | care_point |

#### 解決条件（point scale、半径 6m 基準）

| プラン | 条件 |
|--------|------|
| lighting | 街灯 ×1 |
| hard_fix | 階段 or ハーフ ×1 |
| detour_path | 歩道 3 ノード以上、最長チェーン 2 以上 |
| maintenance | ベンチ ×1 + 歩道 ×2 |
| sign_info | 看板 ×1、または ベンチ + 歩道 |
| care_point | ベンチ ×1 + 街灯 ×1 |
| transit_link | 本島・離島に ferry_dock、航路接続 |

line / area は件数・半径が増加（例: line hard_fix は ×2、area は ×3）。

**実装:** `barrierActions.js`, `barrierValidation.js`

- 写真（ピン付き `PhotoPinSurface`）
- メタデータチップ: scale, factor, needType, 場所型, tag, demographic, timeTag, severity, affectedGroups（ピクトグラム付き）
- 「改善する」→ plan フェーズ
- 削除 → bug 除去、quest を `pending_spawn` に戻す
- 編集（自分の AR 由来 quest のみ）→ `openAREditQuest`

**Phase `plan`:**

- `allowedPlans` からカード選択
- 「DIYを始める」→ `startDIY(bugId, selectedPlan)`

#### DIY 建築モード

| 要素 | 内容 |
|------|------|
| `buildMode` | bugId（または `'free'`） |
| `buildSession` | 改善予算・満足度セッション |
| 上部中央 | **BuildResolutionBanner** — 解決条件達成/未達 |
| 右上 | **ImprovementHudPanel** — 予算・ステークホルダーゲージ |
| 下部 | **ControlBottomBar** — 建築パレット |

- 下部ナビ「フリー建築」→ `startDIY('free')`

### 3.6 完成処理・トレードオフ・島拡張

#### finishBuildMode（`bugSlice.js`）

1. `evaluateActiveBuildResolution` — プラン別 shape 条件
2. `validateFinishSession` — 予算・ブロック上限・満足度
3. bug を `solved: true`
4. リンク quest を `resolved`
5. `postStats` に `resolve` + `build` イベント
6. **トレードオフ:** needType P + plan `hard_fix` → 新 bug「急な勾配で歩きにくい」を offset `[+2.5, 0, +1.5]` に spawn
7. **島拡張** — `computeWorldExpansionAfterSolve`

#### 島拡張ルール（`seaData.js`）

| 条件 | 効果 |
|------|------|
| 解決 1〜3（離島なし） | 本島に **14×14 m** リングチャンク追加 |
| 解決 **≥4** かつ離島なし | 離島 spawn `[96, -0.3, 48]` + transit bug |
| 離島、拡張 lv < 3 | 離島周辺リング拡張 |
| 離島 lv ≥ **3** | 新世代離島（螺旋配置） |

拡張時: ランダム地形ブロック + expansion bug、2.5 秒 god 視点、トースト（`IslandSpawnToast.jsx`）。

#### 解決後の preset 演出

- 該当 quest の expression accent は非表示（`placeNeedTypeStyle.js`）
- 構造的 P/L 系は残る場合あり

#### スポーンルール

| 条件 | L1 場所型 | L2 accent | オーブ |
|------|-----------|-----------|--------|
| archetype 初出 | NEW（base 一式） | NEW | NEW |

```

```

#### スポーン位置（`resolveSpawnPosition`）

1. `mapPin` あればその座標
2. 既存 site があれば anchor 周辺（半径 1.5m、π/3 ずつオフセット）
3. なければ中心からリング探索（±8, ±12 m）、preset 6m / bug 3m 以内回避

## 4. 第三部：AR ↔ ゲームの接続

### 4.1 取り込みフロー

```
Supabase / JSON
  → importArAnnotations()
  → annotationToQuestPost()（Bad のみ、needType 必須）
  → ingestQuestPost()
      → createQuestFromPost()
      → postStats.post イベント
      → spawnQuestOnIsland()（自動）
          → resolveSpawnPosition()
          → buildQuestSpawnBlocks()（L1 + L2）
          → questToBug()
          → questStatus = on_island
```

**重複防止:** `sourceAnnotationId` でスキップ

**手動 fallback:** クエストボード「島に載せる」→ ダブルクリック配置

### 4.2 フィールド対応表

| AR フィールド | ゲームでの使われ方 |
|--------------|-------------------|
| `postKind=bad` | quest / bug として取り込み |
| `needType` | accent 選択、type 写像、オーブ色 |
| `placeArchetype` | L1 場所セット選択 |
| `affectedGroups` | オーブ UI、ステークホルダー満足度初期値 |
| `comment`, `photo`, `photoPins` | BugReportOverlay |
| `worldPin` | メタデータ（将来 GPS 島マップ用） |
| `sourceAnnotationId` | 重複防止、quest ID 安定化（`quest_{id}`） |
| `timeTag`, `severity` | オーブ UI チップ |

### 4.3 エクスポート bridge

- AR: `annotationToGameExport()` — `normalizeAnnotation.js`
- ゲーム: `questImport.js` → `annotationToQuestPost()`
