## タイトル（仮）

**空間注釈と視点置換の連携によるまちづくり合意形成プロセスの提案**


| 語        | 論文中の対応                                          |
| -------- | ----------------------------------------------- |
| 空間注釈     | AR投稿（場所・写真・一言）→ **RQ1**                         |
| 視点置換     | 他者課題をゲーム空間で体験・改善 → **RQ2**                      |
| 連携       | ARで集めた声 → ゲーム課題として載せる                           |
| 合意形成プロセス | WS層がライト層の声を**受容**し、解決に向かう流れ（完全合意の達成まではclaimしない） |


---

## 研究の流れ（全体像）

```
イントロ
  市民共創の進展
  → リビングラボ等の「参加の障壁・固定化」
  → 目的：ライト層の声を集め、まちづくりに取り込む

RQ1：いかにしてライト層の声を取り込むか
RQ2：ワークショップ参加者に、集めたデータをどう受容してもらうか

論理的背景
  ・まちづくり／リビングラボ文脈で何が行われてきたか
  ・市民意見収集手法の比較（評価基準 → 表 → RQ1への含意）

提案・手法
  AR投稿（空間注釈）→ ゲーム（視点置換）→ 体験・評価

結果・考察
  RQ1：入力手段としての妥当性（比較＋必要なら薄い実証）
  RQ2：受容・理解・解決意向など（主評価）
```

---

## 2つのRQ（役割分担）


|              | RQ1               | RQ2                  |
| ------------ | ----------------- | -------------------- |
| **問い**       | ライト層の声をどう取り込むか    | WS参加者が集めたデータをどう受容するか |
| **手段**       | 非同期AR（空間注釈）       | 参加型ゲーム（視点置換）         |
| **検証**       | 文献・比較表が中心。実証は薄くて可 | 小規模ユーザテストが中心         |
| **タイトルとの対応** | 空間注釈              | 視点置換                 |


**重要：** ゲームは「AR促進のため」ではなく、**RQ2（受容）を解く装置**。

---

## 論理の鎖（飛躍を避ける）

### ✅ 正しい順番

```
① 参加障壁・固定化（問題）
② ライト層の声を取り込む（目的）
③ 評価基準を先に決める
④ 既存手法を表で比較
⑤ 入力として AR が「相対的に適している」（RQ1）
⑥ ただし集めるだけでは受容には届かない
⑦ だからゲームで視点置換・体験（RQ2）
```

### ❌ 避ける言い方


| 弱い             | 強い                                |
| -------------- | --------------------------------- |
| ARが良い → だからゲーム | ARは入力に適する → **受容のため**ゲームが必要       |
| ゲームが楽しいから集まる   | 解決でアクセスが広がる（島が育つ）ロジック             |
| ARが効く（断言）      | ARが**目的に照らして適している**（設計 rationale） |
| 合意が formed した  | 受容・理解の**プロセス**を提案・評価              |

## 参加障壁 → 要求（論理の芯）

### 3つの制約（問題）


| 制約      | 内容               | 導かれる要求                   |
| ------- | ---------------- | ------------------------ |
| **時間的** | 都合が合わない          | 非同期性                     |
| **物理的** | 遠い、移動の負担         | 時間・場所への依存の低さ（表では時間的と統合可） |
| **心理的** | 無関係感、少数派なので意味がない | 参加の軽さ（**投稿者側・RQ1**）      |


### 既存手法の限界

- 話し合い中心 → **相手の感情・状況が見えにくい**
- → **感情・状況を記録できる**手法が要る（→ AR / 空間注釈）

### 関係性の課題（RQ2へ）

- 少数派への **哀れみ（pity）** 的な見方
- → **対等な市民としての受容・視点の移動**（哀れみではなく共感的理解）
- → **視点置換（ゲーム／VR）** が担当。**RQ1の表の列では「受容への接続」として1列に留める**

### AR / VR の役割分担


| 手段             | 役割                   | RQ  |
| -------------- | -------------------- | --- |
| **AR**         | 感情・状況の**記録**、軽い非同期入力 | RQ1 |
| **ゲーム（＋VR任意）** | 他者状況の**体験・受容・視点置換**  | RQ2 |


**注意：** 「自分ごと化」は **投稿者（RQ1）** と **受け手（RQ2）** で意味が違う。表では分けて書く。

## 軸選びの思考法（短縮版）

```
① 比較の問いを1文に固定（RQ1）
② 課題 P → Requirement（だから〜が要る）
③ Requirement → 表の列（評価基準）＋ 各列に根拠1行
④ 手法を行に並べ、同じ列で ◎○△
⑤ （任意）デジタル/リアル×受動/能動などは表1の「地図」用。判断は②③の列で行う
```

**評価基準は自分で設定してよい。** 都合が悪く見えない条件：先に目的から導く／全手法を同一基準で見る／他手法の長所・ARの短所も書く／「目的に限れば」と限定する。

## 既存手法比較（RQ1用）

### 要求リスト → 表の列（推奨案：5列）


| 列名                  | 根拠（課題から）       | 測ること                                        |
| ------------------- | -------------- | ------------------------------------------- |
| **A. 時間・場所の制約への対応** | 時間的＋物理的制約      | 非同期か、移動・集合を要しないか（**非同期と場所依存脱却はここで統合**）      |
| **B. 参加の重さ**        | 心理的制約・固定化      | 軽量〜重量（WSほど重くないか）                            |
| **C. 感情・状況の記録**     | 話し合いでは見えにくい    | 写真・場所・文脈・感情が残るか（空間注釈）                       |
| **D. 投稿者の心理的参入**    | 無関係感・少数派感      | 匿名・低負荷・「行っても意味ない」を緩和できるか（**自分ごと化のRQ1側**）    |
| **E. 受容・体験への接続**    | 集めただけでは受容に届かない | RQ2（ゲーム）に載せやすいか（**RQ2への橋。全手法弱ければゲーム提案の根拠**） |


**「関係性」** は列名として広い → **E** に具体化（受容・視点移動への接続）。哀れみ構造への言及は本文・RQ2で。

### 既存枠組み：参加の梯子（Arnstein）との関係

**6列目にはしない。** 梯子が測るのは「市民の決裁への関与度・権力」、本表5列が測るのは「ライト層取り込みへの適合」で **次元が違う**。


| 使い方              | 内容                                                                 |
| ---------------- | ------------------------------------------------------------------ |
| **第2章（背景）**      | 固定化＝梯子の **上位段（協働・協議）に参加資源が集中**し、下位・非参加層が取り残される、と書く                 |
| **表2の脚注 or 補助列** | 各手法の **おおよその梯子位置** を参考情報として1列（**評価には使わない**）                        |
| **考察**           | ARは梯子を一段「上げる」というより、**低い参入コストで Consultation / Informing に近い入力を広げる** |


**Arnstein（1969）8段の略称（引用用）**

1. 操作 2. 治療 3. 情報提供 4. 協議 5. 懐柔 6. 協働 7. 市民管理 …（訳は文献に合わせる）

**手法のざっくり位置づけ（参考・議論用）**


| 手法              | 梯子上的な位置（目安）          | 本表5列との関係             |
| --------------- | -------------------- | -------------------- |
| WS              | 協議〜協働（4〜6）           | B重い・A弱い → 固定化しやすい    |
| みんスタ            | 協議（4）前後              | 制度接続は強いが A・D は中程度    |
| Webアンケート        | 情報提供〜協議（3〜4）         | A良・C弱いことが多い          |
| 参加型GIS          | 協議（4）                | C強いが Bは中〜重           |
| SNS / LINE      | 情報提供〜協議（3〜4）         | A良・Cは中               |
| **非同期AR**       | 協議（4）に**近いが参入は軽い**   | A・B・C・Dで相対的に良い       |
| **本研究ゲーム（RQ2）** | 協議〜協働（4〜6）の**体験的準備** | E列・RQ2。梯子「だけ」では説明しない |


**本文に入れる1文（例）**

> 本節の比較基準は、Arnstein の参加の梯子が示す **参入の程度・権力** とは異なり、ライト層の意見取り込みという **目的適合** に基づく。固定化問題は、梯子の上位手法への集中として理解できる（Arnstein, 1969）。

**他に触れるなら（1つで十分）**

- **IAP2 スペクトラム**（Inform / Consult / Involve / Collaborate / Empower）— 梯子より細かいが、使い方は同様「背景＋参考位置づけ」
- 卒論では **Arnstein 1本＋必要なら IAP2 を脚注** で足りる

**B列「参加の重さ」との違い**

- 梯子 … 市民が **どれだけ決められるか**（権力・段階）
- B列 … その手法に **どれだけ労力が要るか**（負荷）
- 関連はあるが **同じ列にしない**（WSは梯子は高いが、本研究の問題は「高段に来る人が固定化」）

### 自分案6列との対応


| 自分案     | 推奨                                 |
| ------- | ---------------------------------- |
| 非同期     | → **A に統合**                        |
| 場所依存の脱却 | → **A に統合**                        |
| 参加の重さ   | → **B** そのまま                       |
| 自分ごと化   | → **D**（RQ1）と **RQ2本文** に分割        |
| データの記録  | → **C**（「感情・状況の記録」に改名するとARの理由が伝わる） |
| 関係性     | → **E** に具体化                       |


**6列でも書けるが、Aで非同期と場所依存を統合すると5列が読みやすい。** 多すぎるというより **重複が多い** のが問題。

### 先に決める評価基準（旧メモ・参照用）

- 参入障壁の低さ
- 非同期参加の可否
- 現場性（場所・写真・文脈）
- ライト層との適合
- 体験・受容への接続

### 表に載せる候補

- ワークショップ
- みんスタオンライン
- Webアンケート
- 参加型GIS / マップ
- SNS / LINE
- **非同期AR**
- （参考）ゲーム内投稿

### 結論の書き方

> 本研究の目的（リビングラボ非参加者・低負荷・現場性）に照らすと、**非同期AR（空間注釈）が入力手段として相対的に適している**。一方、体験・受容への接続は弱いため、RQ2でゲームを提案する。

## 提案パイプライン

```
[ライト層]  AR投稿（空間注釈）
              ↓ 声・写真・場所
[変換]      クエスト／島上の課題化
              ↓
[WS層等]    ゲーム（視点置換：他者課題を体験）
              ↓ プラン選択・建築・島拡張
[評価]      受容・理解・解決案・意識変容
```

- **島が育つ** ＝ アクセス可能領域の拡大（教授が評価した芯）
- VRヘッドセット：必須ではない。短い没入体験の副次利用 or 展望

## 自分へのメモ

- **風呂敷を広げすぎると混乱する。** 主役は RQ2（受容）+ 比較で支える RQ1（入力）。
- ゲーム作りは「走りすぎ」ではなく、**RQ2の装置として資産**。
- 教授が気に入ったのは **課題→解決→アクセス拡大（島成長）** のロジック。
- VR機材：主評価外でも罪悪感不要。先にスコープを教授と合意。

## 手法の選定基準 — 試行手順（開示用）

> 目的：「代表が恣意的」「この手法はないのはなぜ」への防御。  
> MECEの完全分類ではなく、**スコープ → 候補の洗い出し → 除外の記録 → 代表選定 → 5列評価** の順で再現可能にする。

### Step 0：研究の境界を1段落で固定する（最優先）

**書くこと（チェックリスト）**

- フィールドは **リビングラボ型の市民共創・まちづくり**
- **パブコメ（法定手続）** は対象外とする理由（目的・期間・参加者像が異なる）
- 扱うのは **意見・不満・良い点の「収集」**（決定手続そのものではない）
- 文献 or 事実を1つ以上（LLの固定化、パブコメの位置づけ）

**出力物：** 「研究の境界」段落（200〜400字）— **表より先に書く**

---

### Step 1：「手法」とは何かを定義する（ブレ防止）

本卒論でいう「手法」とは：

> まちづくり・市民共創の文脈で、市民から **不満・意見・良い点** を **集める** ための手段（プロセス・ツール・場の型）

**含まない例（最初から除外候補）**

- 投票・選挙
- 計画の **決定** のみ（パブコメ提出の手続説明だけ）
- 研究者のインタビューのみ（収集「手法」として扱うなら別枠で明記）
- 本研究の **提案ゲーム**（RQ2。表2の行にはしない）

**出力物：** 定義1文 ＋ 除外候補メモ

---

### Step 2：候補を制限なくリストアップ（ブレインストーム）

文献・授業・ゼミ・実例から **名前だけ** 並べる（10〜20個で可）。

例：WS、タウンミーティング、世界カフェ、みんスタ、自治体アプリ、FixMyStreet、PGIS、オープンマップ、SNS、LINE、写真投稿、AR、ストリートビュー、ゲーム内投稿、パブコメ、住民アンケート、プレゼン大会、…

**ルール：** この段階では除外しない。メモ帳・スプレッドシートの「候補」シート。

**出力物：** 候補リスト（番号付き）

---

### Step 3：除外基準（Exclusion）を **先に** 文章化する

候補を減らす **ルール** を、表に載せる前に書く。

**試行案（そのまま使える）**


| #   | 除外基準                                  | 除外される例  |
| --- | ------------------------------------- | ------- |
| E1  | 法定計画手続としての意見提出が主目的                    | パブコメ    |
| E2  | 市民の自由記述・現地文脈の収集が主でない                  | 単純な賛否投票 |
| E3  | リビングラボ／共創場と **比較対象として接続しにくい**         | 純マーケ調査  |
| E4  | 本研究の RQ1（ライト層・低負荷）と **比較の物差しが立てられない** | （該当時のみ） |


**やること：** Step 2 の各候補に E1〜E4 を当てはめ、除外なら **理由1行** を記録。

**出力物：** 「除外一覧」表（候補名｜除外した基準｜1行理由）— **論文の付録 or 脚注にできる**

### Step 4：分類の地図（表1）で残った候補を棚卸し

## 比較表前の要約段落（そのまま使える）

> リビングラボ参加の固定化は、時間的・物理的・心理的限制により生じる。ライト層の意見取り込みには、時間・場所への依存の低さ、参加の軽さ、感情・状況の記録可能性が求められる。一方、話し合い中心の既存手法は他者の状況の共有に限界があり、少数派に対する哀れみ的な見方では対等な市民参加につながりにくい。そこで本研究は、空間注釈（AR）による意見収集と、視点置換（ゲーム体験）による受容の連携を提案する。入力手段の選択は、以下の評価基準に基づく比較により位置づける。

## オニオンモデル実践（Geekiyanage, Fernando & Keraminiyage 2021）

出典：`docs/sustainability-13-08992-v4.pdf`（Sustainability 13:8992）。Figure 8。  
**使い方：** 皮を **2回** むく（RQ1＝consult / RQ2＝involve）→ 候補を表2の行に落とす → A〜Eで採用を決める。

### 皮むき①（RQ1：ライト層の声を取り込む）

| 層 | 本研究での設定 |
|----|----------------|
| **1. 都市開発段階** | **Pre-design / Briefing**（課題発見・意見収集。まちづくりの「企画前〜たたき台」） |
| **2. IAP2** | **Consult**（意見を聞く・フィードバックを集める。empower は狙わない） |
| **3. 目的（論文の12目的から）** | ・地域の課題・不満・希望を**広く聞く** ・固定化の外にいる層の声を**取り込む** ・計画のたたき台に使う**素材**を集める |
| **4. 候補手法（論文34手法から絞り）** | Surveys（Web）／Polls／Community mapping・**PPGIS**／Citizen science・Crowdsourcing（SNS）／Focus groups（重いので代表比較用）／Workshop（同期・重いので対照）／市民参加PF（みんスタ型） |
| **5. ツール例** | Google Forms、Maptionnaire、SNS、自治体PF、**スマホ＋位置＋写真** |
| **6. 技法** | 非同期投稿、低負荷、現地から一言＋写真 |

→ **表2へ：** 上記のうち代表6行（WS, みんスタ, PGIS, SNS, Webアンケート, **非同期AR**）を A〜E で比較。

→ **採用（表2後）：** 入力目的に限り **非同期AR（空間注釈）**。根拠：A・B・Cがバランス良い。Eは△。

### 皮むき②（RQ2：集めた声を受容する）

| 層 | 本研究での設定 |
|----|----------------|
| **1. 都市開発段階** | **Briefing〜Professional design**（たたき台に市民の声を載せ、WSで咀嚼する段階） |
| **2. IAP2** | **Involve**（意思決定に招き、複数回フィードバック。collaborate まではclaimしない） |
| **3. 目的** | ・他者（ライト層）の課題・状況を**理解する** ・哀れみではなく**対等な受容** ・解決・改善の**体験** |
| **4. 候補手法（論文から）** | Workshop／Community mapping（共同）／**Serious games**（論文 case D: NextCampus）／Design charrette／GMB・SD（重い）／Citizen juries（重い） |
| **5. ツール** | 3Dシミュレーション、ストーリー、バーチャル歩行、**本研究＝ブラウザ参加型ゲーム** |
| **6. 技法** | 視点置換、課題→プラン→建築→島拡張（アクセシビリティ） |

→ **表2の E列△の説明：** consult 系（①）では受容に届かない → involve 系（②）が要る。

→ **採用：** **Urban Alchemist（参加型ゲーム）**。VRは必須ではない（副次可）。

### 2回皮むきから言える結論（1段落）

> 本研究は Geekiyanage et al. (2021) のオニオンモデルに沿い、意見収集（RQ1）を consult レベル・企画前段階の参加として位置づけ、他者意見の受容（RQ2）を involve レベルの体験として位置づけた。前者の候補を本研究の評価基準（A〜E）で比較した結果、非同期の空間注釈（AR）を入力手段に採用する。後者については consult 系手法では受容への接続が限られるため、視点置換を担う参加型ゲームを組み合わせる。

### 文献メモ（オニオンで言えることだけ）

- 実務は inform・consult に偏り、involve・collaborate は少ない → **2段パイプラインの根拠**
- 1プロジェクトで IAP2 の5段全部は推奨されない → **RQ1とRQ2を分ける根拠**
- Serious games は involve 事例に含まれる → **ゲームは「何でもよい」ではなく involve 系の文献上の候補**
