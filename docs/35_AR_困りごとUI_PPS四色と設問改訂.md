## 1. 学術的根拠（PDF 要約）

### 1.1 PPS Place Diagram — 4 属性

[Project for Public Spaces (PPS)](https://www.pps.org/article/grplacefeat) の **Place Diagram** を最上位概念に、市民の不満を「4 属性のいずれかが阻害された状態」と定義。

| # | PPS 属性 | 本システムでの呼び方 | UI 色（案） |
|---|----------|----------------------|-------------|
| ① | Access & Linkages | **動**（移動の壁） | 🟦 `#3949ab` |
| ② | Uses & Activities | **静**（滞在の壁） | 🟩 `#388e3c` |
| ③ | Comfort & Image | **環境**（五感の壁） | 🟨 `#f9a825` |
| ④ | Sociability | **社会**（関係の壁） | 🟧 `#ef6c00` |

### 1.2 日本のガイドライン → 8 類型

PDF のマッピング（卒論引用用）:

| PPS | ガイドライン例 | UI 8 類型 |
|-----|----------------|-----------|
| 動 | 国交省バリアフリー GL、街路交通 GL、ウォーカブル GL | 歩きにくい / 行き来しにくい / 迷いやすい |
| 静 | ウォーカブル GL（ほこみち・ベンチ） | 休みにくい |
| 環境 | CPTED（死角）、割れ窓・維持管理 | 見えにくい / 汚れ・荒れ |
| 社会 | CPTED（心理的治安）、地域福祉 GL | こわい・不安 / 頼れる場がない |

**needType コードとの対応（既存維持）:**

| グループ | needType | ラベル |
|----------|----------|--------|
| 動 | P | 歩きにくい |
| 動 | L | 行き来しにくい |
| 動 | I | 迷いやすい |
| 静 | R | 休みにくい |
| 環境 | V | 見えにくい |
| 環境 | M | 汚れ・荒れ |
| 社会 | S | こわい・不安 |
| 社会 | C | 頼れる場がない |

### 2.1 UI ルール

| ルール | 内容 |
|--------|------|
| 選択 | **1 つのみ**（ラジオ）。タップで `needType` 更新 |

| 見えにくい | 暗い・死角で**見えない** |
| こわい・不安 | 人通り・雰囲気で**安心できない**（暗さ以外も） |

## 4. 「誰にとって」— チップ + **その他（記述式）**

### 4.1 結論

| 要素 | 設計 |
|------|------|
| チップ | **6 つ・すべて任意**（0 個でも次へ進める） |
| **その他** | チップ 1 つ。**選択時だけ** 記述欄を表示（最大 30 字） |
| 常時表示の「ほか」欄 | **廃止**（その他チップに統合） |
| time / severity | 折りたたみ任意（デフォルト: 常時 / 中） |

**高齢者:** 「歩行がゆっくりな人」ではなく **「高齢者」** ラベルのまま（既存 pictogram・RQ2 互換）。

### 4.2 チップ（確定案）

| id | ラベル | 備考 |
|----|--------|------|
| `all` | みんな | 選択時は他チップ解除 |
| `senior` | 高齢者 | 現行 `TARGET_GROUP_OPTIONS` を維持 |
| `wheelchair` | 車いす | 現行維持 |
| `stroller` | ベビーカー | 現行維持 |
| `child` | 子ども | 現行維持 |
| `other` | **その他** | タップで記述欄表示 |

**観光客:** 固定チップからは外す（**その他** に「観光客」等を書く）。

保存形式:

```javascript
affectedGroups: ['高齢者', '車いす'],  // その他以外
affectedOther: 'チャリで通る人',        // その他選択時のみ（payload）
// affectedGroups に「その他」ラベルも入れてよい: ['その他'] + affectedOther
```

### 4.3 UI（ステップ 3）

```
誰にとって？（任意・複数可）

[みんな] [高齢者] [車いす] [ベビーカー] [子ども] [その他]

　↑「その他」選択時のみ ↓
┌──────────────────────────────┐
│ 例：チャリ、観光客、夜勤の人    │  ← 30字以内
└──────────────────────────────┘

▼ 詳しく（任意）
  いつ？ … デフォルト「常時」
  どのくらい？ … デフォルト「中」
```

**その他の挙動:**

- 「その他」ON → 記述欄表示。**1 文字以上** ないと「その他」だけでは不完全（任意: その他 OFF に戻す or 警告）
- 「みんな」選択 → その他含め他チップすべて解除

## 5. 実装と Supabase — 何が先か

| 機能 | Supabase 必須？ | 備考 |
|------|-----------------|------|
| PPS 四色 needType UI（PR-C） | **不要** | フロントのみ |
| 誰にとって + その他（PR-D） | **不要** | payload に保存。未設定時は端末内 |
| 投稿の **端末間同期** | **必要** | 既存 `ar_annotations` |
| いいね + **共感ランキング**（PR-E） | **必要** | `ar_likes` + 全員分の集計 |
| Vercel 公開 URL | Supabase と **セット推奨** | アプリだけ載せても同期なし |

**推奨順:**

```
1. Supabase 設定（31 / 32）— 街歩き・複数人を想定するなら早めに
2. PR-C, PR-D — Supabase なしでも開発・単体テスト可
3. migration 003 + PR-E — Supabase 接続後に結合テスト
4. Vercel デプロイ — 本番 URL
```

**全部の前提が Supabase というわけではない。** UI 改修は先に進められる。  
**共感ランキングまで含めた「完成形」** には Supabase が必須。
