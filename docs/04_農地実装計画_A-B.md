## 1. 目的と位置づけ

### 1.1 ゲーム上の目的

Urban Alchemist は「まちづくり × 住民の想像力」を核とする。農地は次の体験を足す。

- **暮らしの土台**：食・風景・地域性を、プレイヤーが地面として表現できる
- **歩く・集う場の拡張**：広場・歩道の隣に「畑」「田んぼ」を置いて、まちの物語を厚くする
- **将来拡張の土台**：季節（青田・黄金色）や島拡張地形とつなぎやすいデータ構造にする

### 1.2 本計画でやること / やらないこと

| やる（A〜B） | やらない（別フェーズ） |
|-------------|----------------------|
| 畑・田んぼ・家庭菜園枠の**地面タイル配置** | 栽培フェーズ（種まき→成長→収穫） |
| 土色・水色などの**自由色選択**（植物と同型） | 収穫ゲーム・クエスト連動 |
| 隣接タイルで畝・畦が**つながって見える**（Bライト） | 内角・外角の完全な曲線畝 |
| Undo / セーブ / ゴーストプレビュー | 島拡張時の滝・山・池（Phase 4） |
| 建築パレットの「農地」モード | 昼夜・季節（Phase 5） |

## 2. スコープ定義

### Phase 3-A：農地タイル（必須）

**ゴール**：プレイヤーが「畑」「田んぼ」を1マス単位で置き、色を変えられる。

| 形状ID | 表示名 | 概要 |
|--------|--------|------|
| `farm_plot` | 畑 | 土の床＋畝（単独でも成立する簡易メッシュ） |
| `rice_paddy` | 田んぼ | 畦で囲んだ水面＋土縁 |
| `garden_bed` | 菜園・花壇枠 | 低い枠＋土（畑より小さく装飾寄り） |

**配置ルール**

- 既存ブロックと同様：`pos` はグリッドスナップ（0.5m 単位）
- Y オフセット：`path` / `turf` と同様、床面に沈めて配置（例: `y - 0.24` 付近）
- 素材：`material` は `grass` または `sand` をデフォルト（見た目は `agri.color` 優先）
- スケール：初期は `[1, 1, 1]` 固定推奨（ギズモ拡張は後回し可）

### Phase 3-B ライト：隣接連続見た目（必須・Aと同一リリース可）

**ゴール**：同種の農地タイルが隣接すると、**辺単位**で畝・畦が共有され、バラバラなタイルに見えない。

| 対象形状 | 連続ルール（ライト版） |
|----------|------------------------|
| `farm_plot` | 隣接辺（+X, -X, +Z, -Z）に同じ `farm_plot` がある場合、その辺の畝ラインを非表示または薄くする |
| `rice_paddy` | 隣接辺に同じ `rice_paddy` がある場合、共有辺の畦を1本化（重複畦を出さない） |
| `garden_bed` | Bでは隣接連動なし（単体装飾のまま） |

**Bライトで後回しにするもの**

- L字・凹型の内角で畝が曲がる表現
- 斜め隣接（対角）の判定
- 範囲選択による一括「大畑」メッシュ生成
- 高さ差による水位の変化

## 3. 技術方針

### 3.1 設計原則

1. **`App.jsx` にロジックを足さない** — 定数・メッシュ・パレット・隣接ユーティリティを分割
2. **植物実装（`nature/`）のパターンを踏襲** — `Block.jsx` 委譲 + `useGameStore` 拡張
3. **農地データは `agri` フィールドに集約** — `nature` と混在させない
4. **Bの隣接判定は純関数** — テストしやすく、描画コンポーネントから切り離す

### 3.2 アーキテクチャ概要

```
ControlBottomBar
  └─ paletteMode: 'blocks' | 'nature' | 'agri'
       └─ AgriPalette → setSelectedShape + setSelectedAgriColor

useGameStore
  └─ placeBlockAtHover → placedBlocks[].agri = { color }

MainGameScene
  └─ placedBlocks + getAgriNeighbors(block, allBlocks)
       └─ Block → AgriMeshes(neighbors)

Block.jsx
  └─ isAgriShape(shape) → <AgriMeshes />
```

## 4. データ仕様

### 4.1 定数（`src/constants/agriData.js`）

```javascript
export const AGRI_SHAPES = ['farm_plot', 'rice_paddy', 'garden_bed'];

export const AGRI_COLORABLE_SHAPES = ['farm_plot', 'rice_paddy', 'garden_bed'];

export const DEFAULT_AGRI_COLORS = {
  farm_plot:   '#8d6e63',  // 土
  rice_paddy:  '#4fc3f7',  // 水
  garden_bed:  '#a1887f',  // 菜園土
};

export const AGRI_META = {
  farm_plot:   { label: '畑',     icon: '🌾', defaultMaterial: 'sand',  defaultScale: [1,1,1] },
  rice_paddy:  { label: '田んぼ', icon: '🍚', defaultMaterial: 'water', defaultScale: [1,1,1] },
  garden_bed:  { label: '菜園',   icon: '🥬', defaultMaterial: 'grass', defaultScale: [1,1,1] },
};
```

### 4.2 配置ブロック拡張

既存 `placedBlocks` エントリに optional フィールドを追加：

```json
{
  "id": "…",
  "pos": [0, 0.25, 0],
  "shape": "farm_plot",
  "material": "sand",
  "rotation": 0,
  "scale": [1, 1, 1],
  "agri": {
    "color": "#8d6e63"
  }
}
```

- `agri.color`：hex 文字列（`#RRGGBB`）
- 将来 `agri.seasonVariant` などを足せる余地を残す
- **後方互換**：`agri` 無しのブロックは `DEFAULT_AGRI_COLORS[shape]` を使用

### 4.3 Zustand 追加状態

| 状態 | 型 | 用途 |
|------|-----|------|
| `selectedAgriColors` | `Record<shape, hex>` | 形状ごとの現在色 |
| `setSelectedAgriColor` | `(shape, color) => void` | パレットから色更新 |

`setSelectedShape(shape)` 時：

- `isAgriShape(shape)` なら `AGRI_META[shape].defaultScale` / `defaultMaterial` を適用
- `isNatureShape(shape)` との排他は UI タブで担保（ストア上は最後に選んだ shape が有効）

## 5. 3D メッシュ仕様（Phase 3-A）

### 5.1 共通

- 基準サイズ：1マス = 0.5 × 0.5（既存ブロックと統一）
- ゴースト：半透明（植物と同様 `opacity ≈ 0.55`）
- ホバー：削除=赤 / 編集=黄 / 建築中=シアン
- 物理：`CuboidCollider` 薄型（歩行可能、高さ ≈ 0.02〜0.05）

### 5.2 `farm_plot`（畑）

| パーツ | ジオメトリ | 備考 |
|--------|-----------|------|
| 土台 | Box 0.5 × 0.02 × 0.5 | `agri.color`、Y = -0.24 |
| 畝（東西2本） | Box 細長 | 単体時は常に表示（Bで隣接辺を抑制） |

### 5.3 `rice_paddy`（田んぼ）

| パーツ | ジオメトリ | 備考 |
|--------|-----------|------|
| 水面 | Box 0.42 × 0.015 × 0.42 | `agri.color`、やや透過可 |
| 畦（4辺） | Box 細長 | 高さ 0.03、土色 `#6d4c41` 固定でも可 |
| 角 | 小さめ Box | 単体時の閉じた印象 |

### 5.4 `garden_bed`（菜園）

| パーツ | ジオメトリ | 備考 |
|--------|-----------|------|
| 枠 | Box 0.5 × 0.06 × 0.5 | 木色 or `agri.color` の暗め |
| 土 | Box 0.42 × 0.02 × 0.42 | 内側 |

```javascript
  // 条件: 同じ shape、|Δpos| < 0.05、Y は同一層とみなす（|Δy| < 0.1）
}
```

### 6.2 描画への反映

`AgriMeshes` に `neighbors` prop を渡す。

**farm_plot**

| 辺 | 隣に farm_plot あり | 描画 |
|----|---------------------|------|
| +X | true | +X 側の畝を非表示 |
| -X | true | -X 側の畝を非表示 |
| ±Z | 同様 | 同様 |

**rice_paddy**

| 辺 | 隣に rice_paddy あり | 描画 |
|----|----------------------|------|
| 共有辺 | true | その辺の畦メッシュを1ブロック側のみ描画（重複防止） |

### 6.3 更新トリガー

隣接見た目が変わるタイミング：

- 農地ブロックの**新規配置**
- **消去**（eraser）
- **Undo / Redo**
- 範囲削除（将来）

→ `placedBlocks` が変われば React の再レンダーで足りる（専用イベント不要）。

### 6.4 MainGameScene での渡し方

```javascript
placedBlocks.map(block => {
  const neighbors = isAgriShape(block.shape)
    ? getAgriNeighbors(block, placedBlocks)
    : null;
  return (
    <Block
      …
      agri={block.agri}
      agriNeighbors={neighbors}
    />
  );
});
```

`Block.jsx` は `agriNeighbors` を `AgriMeshes` にそのまま委譲。

## 7. UI 仕様

### 7.1 パレット構成（ControlBottomBar）

既存の `paletteMode` を拡張：

| モード | 内容 |
|--------|------|
| `blocks` | 既存の建築・デコ・ツール |
| `nature` | 既存の植物（NaturePalette） |
| `agri` | 新規 AgriPalette |

### 7.2 AgriPalette

植物パレットと同型：

1. 形状ボタン（畑 / 田んぼ / 菜園）
2. 選択中形状のみ色 UI を表示
   - `<input type="color">`
   - 色相スライダー（0〜360）
   - 土系・水系のプリセットスウォッチ（農地向け）

**プリセット例（farm_plot / garden_bed）**  
茶色、赤土、黒土、砂色

**プリセット例（rice_paddy）**  
青水、浅緑水、泥水色

### 7.3 ガイダンス文（フッター）

農地選択時：

> 🌾 【地面をクリック】で農地を配置します

## 11. 工数・リスク

### 11.1 工数感（目安）

| 塊 | 目安 |
|----|------|
| 3-A 全体 | 0.5〜1.5 日（植物実装経験者） |
| 3-B ライト | 0.5〜1 日 |
| 仕様書追記・微調整 | 0.25 日 |

### 11.2 リスクと対策

| リスク | 対策 |
|--------|------|
| 畝・畦の Z-fighting | 共有辺メッシュをわずかに低くする（Y -0.001） |
| 隣接判定の誤検知（Y差） | `|Δy| < 0.1` で同一層に限定 |
| `nature` / `agri` の混同 | shape ID とフィールド名を厳密分離 |
| パレットが長くなる | 3モードタブでスクロール量を抑える |
| パフォーマンス（全ブロック隣接探索 O(n²)） | 現状の placedBlocks 規模なら可。将来は pos キーの Map 化 |
