# RQ1 AR投稿アプリ v2

東京都江東区向け **GPS＋コンパス AR** 投稿システム。ゲーム（RQ2）とは `/ar.html` で分離。

## 起動（Mac 単体）

```bash
npm run ar:server          # LAN 同期（任意）
VITE_AR_API_URL=http://localhost:8787 npm run dev
# またはクラウド同期: .env.local に VITE_SUPABASE_* → npm run dev
```

- ARアプリ: https://localhost:5173/ar.html

## スマホで触る（現地 AR・コンパス用）

Mac の `localhost` はスマホからは見えません。**同一 Wi‑Fi** で Mac の IP 経由で開きます。

### 1. ターミナル A — 同期サーバー

```bash
npm run ar:server
```

### 2. ターミナル B — 開発サーバー（LAN 公開 + API プロキシ）

```bash
npm run dev:mobile
```

起動ログに `Network: http://192.168.x.x:5173/` のような行が出ます。

### 3. Mac の IP を確認（ログに出ない場合）

```bash
ipconfig getifaddr en0
```

（Wi‑Fi 以外なら `en1` など）

### 4. スマホの Safari で開く（必ず https）

```
https://192.168.x.x:5173/ar.html
```

**http:// ではカメラ・コンパスは動きません**（ブラウザがブロックします）。

初回は「この接続ではプライバシーが保護されません」→ **詳細 → アクセス** で進んでください。

Google アプリ等の**アプリ内ブラウザ**では動かないことがあります。URL を **Safari** にコピーして開いてください。

### よくあるつまずき

| 症状 | 原因 | 対処 |
|------|------|------|
| サーバに接続できない | スマホで `localhost` を開いている | Mac の `192.168.x.x` を使う |
| 同上 | Mac とスマホが別 Wi‑Fi | 同じ Wi‑Fi に揃える |
| 同上 | Mac ファイアウォール | システム設定 → ネットワーク → ファイアウォールで Node/Vite を許可 |
| カメラ・コンパスが動かない（iPhone） | **http://** で開いている | **https://** の URL を使う |
| 同上 | Google アプリ内で開いている | Safari に URL をコピー |

**iPhone** ではカメラ・コンパスに **HTTPS** が必要なことがあります。その場合は [ngrok](https://ngrok.com/) などで HTTPS トンネルを張るか、開発用に `mkcert` で証明書を入れて `vite` を HTTPS 起動してください。

### Mac でコンパスが使えない理由

**端末の問題**です（設定ミスではありません）。Mac のブラウザは向きセンサー（コンパス）を Web AR 用に提供しません。現地 AR は **スマホ（iPhone / Android）** で試してください。Mac では地図モード・投稿フォーム・記録図鑑は利用できます。

## 画面構成

- **ホーム** — 現地AR / 新規記録 / 地図 / 一覧 から選択（下部タブなし）
- 各画面は **← ホーム** で戻る
- 投稿中は地図・一覧へ切り替え不可

## ピンの刺し方 vs 地図

| 画面 | 役割 |
|------|------|
| **現地AR（＋投稿）** | カメラを向け**画面タップ**で「あそこ」を指定 → GPS+方角から**ワールド座標**を計算して刺す |
| **地図** | 刺さったピンを**俯瞰で確認**。地図上では新規投稿しない |

## 体験フロー

1. 現地AR — 近くのピンをカメラ上に表示
2. ＋ — タップで場所 → 撮影 → 質問2画面 → 刺さる
3. 地図 — 江東区内 OSM、全員のピン位置確認
4. 一覧 — JSON書き出し（RQ2接続用）

## エリア

`src/ar/constants/kotoArea.js` — 江東区ポリゴン近似の bounds。

## 注意

- 屋外・HTTPS（または localhost）でカメラ・コンパス・GPS
- GPS精度によりピン位置は数m〜数十mずれることがある
